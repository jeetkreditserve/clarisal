# Combined Remediation Plan For The HRMS Audit Findings

## What This Plan Is For
This plan is for addressing the issues raised in these two audit reports:

- [HRMS_AUDIT_REPORT_20260402_1711.md](/home/jeet-padhya/PycharmProjects/clarisal/docs/HRMS_AUDIT_REPORT_20260402_1711.md)
- [HRMS_AUDIT_REPORT_20260402_1716.md](/home/jeet-padhya/PycharmProjects/clarisal/docs/HRMS_AUDIT_REPORT_20260402_1716.md)

The goal is to turn those audit findings into a **working action checklist** that can be used during implementation. As each task is finished, it should be marked as done.

This plan is written for a non-technical reader first. It explains:

- what must be fixed immediately
- what can wait for the next sprint or two
- what belongs in the longer roadmap
- how to track progress in a simple checkbox format

## How To Use This Plan
- [ ] Leave a task as `- [ ]` while it is not complete

- [ ] Progress has been tracked in this checklist as work was completed

## Phase 4 - Repair Payroll So It Is Safe For Real Use
Done means: payroll is either narrowed to a safe scope or improved enough that it no longer produces obviously incorrect outcomes.

### 4.1 Fix Payroll Calculation Correctness
- [-] Review how tax is calculated today
- [-] Add missing India payroll logic called out in the audits:
  - [-] ESI (auto-calculated: 0.75% employee, 3.25% employer for gross ≤ ₹21,000)
  - [-] Professional Tax (Maharashtra slab, stored per location state)
  - [-] TDS depth (standard deduction applied, cess added)
  - [-] cess (4% health & education cess on income tax)
  - [ ] surcharge where applicable (>₹50L income, low priority for SMBs)
  - [-] standard deduction where applicable (₹75,000 deducted before tax)
- [-] Fix PF so it is not treated as a flat manual number where it should be formula-driven
- [-] Fix pro-rated salary for joining month
- [-] Fix pro-rated salary for exit month
- [-] Connect approved LOP leave to payroll deductions
- [-] Confirm employer contributions are handled correctly for reporting even if they do not affect take-home pay

### 4.2 Fix Payroll Scope Gaps

- [ ] Add payroll reports needed for operational use
- [ ] Add Form 16 support if the product is meant to support real India payroll
- [ ] Add investment declaration handling if TDS is expected to be accurate
- [ ] Add arrears and revision handling if salary changes are meant to be operationally safe
- [ ] Add full & final settlement if employee exits are meant to be supported properly

- [x] Add better exception handling during payroll processing
